# SECURITY

# Living Civilization Framework (LCF)

## Security Policy

---

## Purpose

The Living Civilization Framework (LCF) is primarily a documentation and research project.

While the repository currently focuses primarily on documentation and conceptual frameworks rather than executable software, maintaining the integrity, security, and trustworthiness of the project remains important.

This document explains how to report potential security-related concerns.

---

## Supported Versions

Security-related reports are generally reviewed for the latest publicly available version of the framework.

Older versions may not receive updates unless the issue significantly affects the project's integrity or documentation.

| Version | Supported |
|---------|:---------:|
| Latest Release | ✅ |
| Previous Releases | ⚠️ Limited |
| Archived Versions | ❌ |

---

## Reporting Security Issues

If you discover a security-related issue, please report it responsibly rather than disclosing it publicly before it can be reviewed.

Examples may include:

- Repository compromise
- Unauthorized modifications
- Malicious pull requests
- Supply chain concerns
- Copyright or licensing issues affecting repository integrity
- Security concerns related to future software components

Please use GitHub's available communication channels, such as private security reporting (if enabled) or by contacting the project maintainers directly.

When reporting an issue, please include:

- A clear description of the concern
- Steps to reproduce (if applicable)
- Relevant evidence or references
- The affected files or components

---

## Scope

At the current stage of development, LCF primarily consists of documentation and conceptual frameworks.

Security reports are most likely to involve:

- Repository integrity
- Documentation authenticity
- Contributor account compromise
- Supply chain or dependency concerns (if technical components are added)
- Licensing or attribution issues affecting project integrity
- Future software, APIs, automation tools, or other technical components, if introduced

---

## AI-Assisted Content

Some documentation has been developed with AI assistance.

If you identify content that may introduce security, legal, attribution, or integrity concerns, responsible reports are welcome.

Contributors remain responsible for reviewing AI-assisted content before publication.

---

## Responsible Disclosure

Please avoid publicly disclosing potential security issues before maintainers have had a reasonable opportunity to review them.

Responsible disclosure helps protect contributors, users, and the integrity of the project.

---

## Continuous Improvement

Like the Living Civilization Framework itself, this Security Policy may evolve as the project grows.

Future versions may expand this policy if software, APIs, automation tools, websites, or other technical components are added to the repository.
